# TaskFlow 정보 아키텍처 (IA)

## 1. 사이트맵 (사이트맵)

```
TaskFlow 웹 대시보드
├── / (홈/칸반 보드)
├── /graph (의존성 그래프)
├── /timeline (타임라인 차트)
├── /settings (설정)
└── /api (API 엔드포인트)
    ├── /api/tasks
    ├── /api/projects
    ├── /api/sse
    └── /api/cli-sync
```

## 2. 사용자 흐름 (사용자 흐름)

### **핵심 태스크 1: 프로젝트 초기 설정 및 PRD 분해**
1. 사용자가 CLI에서 `task init` 실행
2. 웹 브라우저에서 localhost:4000 접속
3. 프로젝트 설정 페이지에서 PRD 파일 경로 지정
4. `task parse-prd` 실행 후 대시보드 새로고침
5. 칸반 보드에서 자동 생성된 태스크들 확인

### **핵심 태스크 2: 태스크 상태 관리 및 실시간 동기화**
1. 칸반 보드에서 태스크 카드 클릭
2. 태스크 상세 드로어 열림
3. 상태 변경 버튼 클릭 (Pending → In-Progress)
4. SSE를 통해 실시간으로 칸반 보드 업데이트
5. CLI에서도 `task list`로 변경된 상태 확인

### **핵심 태스크 3: 요구사항 변경 및 영향 분석**
1. 사용자가 CLI에서 `task refine` 실행
2. AI가 영향받는 태스크들 분석 및 제안
3. 웹 대시보드의 의존성 그래프에서 영향 범위 시각화
4. 변경된 태스크들이 칸반 보드에서 하이라이트
5. 타임라인 차트에서 일정 변경 사항 확인

## 3. 네비게이션 구조 (네비게이션 구조)

### **고정 사이드바 네비게이션 (240px 너비)**
- **로고 영역**: TaskFlow 브랜딩 + 프로젝트 선택 드롭다운
- **주요 메뉴**:
  - 칸반 보드 (G+K 단축키)
  - 의존성 그래프 (G+D 단축키)
  - 타임라인 (G+T 단축키)
  - 설정 (G+S 단축키)
- **하단 영역**: CLI 연결 상태, 버전 정보

### **상단 명령 바**
- 전역 검색/명령 팔레트 (/ 키로 활성화)
- 빠른 태스크 추가 버튼
- 테마 토글 (다크/라이트 모드)

### **모바일 네비게이션 (768px 미만)**
- 햄버거 메뉴로 사이드바 토글
- 하단 탭 네비게이션 (칸반, 그래프, 타임라인)

## 4. 페이지 계층 구조 (페이지 계층 구조)

```
/ (Depth 1) - 칸반 보드 메인 페이지
├── /graph (Depth 1) - 의존성 그래프
├── /timeline (Depth 1) - 타임라인 차트
├── /settings (Depth 1) - 설정
│   ├── /settings/project (Depth 2) - 프로젝트 설정
│   ├── /settings/preferences (Depth 2) - 사용자 환경설정
│   └── /settings/cli (Depth 2) - CLI 연동 설정
└── /task/:id (Depth 1) - 태스크 상세 (모달/드로어)
```

## 5. 콘텐츠 구성 (콘텐츠 구성)

| 페이지 | 주요 콘텐츠 요소 |
|---|---|
| 칸반 보드 (/) | 4개 컬럼 (Pending, In-Progress, Blocked, Done), 태스크 카드, 드래그 앤 드롭 영역, 태스크 상세 드로어 |
| 의존성 그래프 (/graph) | D3 기반 방향 그래프, 노드 (태스크), 엣지 (의존성), 사이드 패널 (선택된 노드 정보) |
| 타임라인 (/timeline) | 간트 차트 형태 바, 마일스톤 마커, 필터 컨트롤, 인사이트 패널 (평균 사이클 타임) |
| 설정 (/settings) | 프로젝트 정보 폼, 테마 선택, 키보드 단축키 참조, CLI 버전 상태 |
| 태스크 상세 | 메타데이터 (ID, 상태, 복잡도), 마크다운 설명, 액션 버튼 (상태 변경, 브레인스토밍) |

## 6. 인터랙션 패턴 (인터랙션 패턴)

### **칸반 보드 인터랙션**
- **드래그 앤 드롭**: dnd-kit 라이브러리 사용, 컬럼 간 태스크 이동 시 자동 상태 업데이트
- **카드 클릭**: 우측 드로어에서 태스크 상세 정보 표시
- **키보드 네비게이션**: 화살표 키로 카드 간 이동, Enter로 상세 보기

### **명령 팔레트**
- **트리거**: `/` 키 또는 Cmd+K (Mac), Ctrl+K (Windows)
- **기능**: 페이지 이동, 태스크 검색, 빠른 액션 실행
- **자동완성**: 타이핑 시 실시간 필터링

### **실시간 업데이트**
- **SSE 스트림**: CLI 변경사항을 300ms 이내 웹 UI에 반영
- **낙관적 UI**: 사용자 액션 즉시 UI 업데이트, 실패 시 롤백

### **모달 및 확인 대화상자**
- **태스크 삭제**: 확인 모달 표시
- **벌크 액션**: 다중 선택 후 일괄 상태 변경
- **ESC 키**: 모든 오버레이 닫기

## 7. URL 구조 (URL 구조)

### **기본 규칙**
- **루트**: `/` (칸반 보드)
- **기능별**: `/feature-name`
- **설정 하위**: `/settings/sub-category`
- **동적 라우팅**: `/task/:taskId` (태스크 상세)

### **구체적 URL 매핑**
```
/ → 칸반 보드 메인
/graph → 의존성 그래프
/timeline → 타임라인 차트
/settings → 설정 메인
/settings/project → 프로젝트 설정
/settings/preferences → 사용자 환경설정
/settings/cli → CLI 연동 설정
/task/001 → 태스크 001 상세 (모달)
```

## 8. 컴포넌트 계층 구조 (컴포넌트 계층 구조)

### **글로벌 컴포넌트**
- **AppLayout**: 전체 레이아웃 래퍼
- **Sidebar**: 고정 사이드바 네비게이션
- **CommandPalette**: 전역 명령 팔레트
- **Toast**: 알림 시스템
- **LoadingSpinner**: 로딩 상태 표시

### **페이지별 컴포넌트**

#### **칸반 보드 페이지**
- **KanbanBoard**: 메인 칸반 컨테이너
  - **KanbanColumn**: 개별 컬럼 (Pending, In-Progress 등)
    - **TaskCard**: 태스크 카드 아이템
      - **StatusBadge**: 상태 표시 뱃지
      - **ComplexityTag**: 복잡도 태그
  - **TaskDetailDrawer**: 태스크 상세 드로어
    - **TaskMetadata**: 메타데이터 표시
    - **MarkdownRenderer**: 설명 렌더링
    - **ActionButtons**: 액션 버튼 그룹

#### **의존성 그래프 페이지**
- **DependencyGraph**: D3 그래프 컨테이너
  - **GraphNode**: 개별 태스크 노드
  - **GraphEdge**: 의존성 연결선
- **GraphSidePanel**: 선택된 노드 정보 패널

#### **타임라인 페이지**
- **TimelineChart**: 메인 타임라인 컨테이너
  - **GanttBar**: 개별 태스크 바
  - **MilestoneMarker**: 마일스톤 표시
- **TimelineFilters**: 필터 컨트롤
- **InsightsPanel**: 통계 및 인사이트

#### **설정 페이지**
- **SettingsLayout**: 설정 페이지 레이아웃
  - **SettingsTabs**: 탭 네비게이션
  - **ProjectSettings**: 프로젝트 설정 폼
  - **PreferencesSettings**: 사용자 환경설정
  - **CLISettings**: CLI 연동 설정

### **공통 UI 컴포넌트**
- **Button**: 기본 버튼 (Primary, Secondary, Ghost)
- **Input**: 입력 필드
- **Select**: 드롭다운 선택
- **Modal**: 모달 대화상자
- **Card**: 카드 컨테이너
- **Badge**: 상태/태그 뱃지
- **Tooltip**: 툴팁 도움말

### **데이터 흐름 컴포넌트**
- **SSEProvider**: Server-Sent Events 컨텍스트
- **QueryProvider**: React Query 설정
- **ThemeProvider**: 테마 관리
- **CLISyncProvider**: CLI 동기화 상태 관리

## 9. 상태 관리 및 데이터 흐름

### **데이터 소스**
1. **로컬 마크다운 파일** (.taskflow/ 디렉터리)
2. **Hono API 라우트** (서버 액션)
3. **React Query 캐시** (클라이언트 상태)
4. **SSE 스트림** (실시간 업데이트)

### **상태 업데이트 플로우**
```
CLI 명령 실행 → 마크다운 파일 수정 → 파일 시스템 감시 → SSE 이벤트 발송 → 웹 UI React Query 무효화 → 컴포넌트 리렌더링
```

### **오프라인/에러 처리**
- **낙관적 업데이트**: 사용자 액션 즉시 UI 반영
- **에러 롤백**: API 실패 시 이전 상태로 복원
- **재시도 메커니즘**: 네트워크 오류 시 자동 재시도

본 정보 아키텍처는 TaskFlow의 웹 대시보드와 CLI 도구 간의 원활한 상호작용을 지원하며, 1인 개발자의 효율적인 프로젝트 관리를 위한 직관적이고 전문적인 사용자 경험을 제공하도록 설계되었습니다.
